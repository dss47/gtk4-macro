\section{Introduction}
Ce rapport documente l'implémentation interne du wrapper GTK4. Contrairement au premier volume, centré sur les structures de configuration et l'usage de l'API, ce second document décrit la logique des fonctions, les appels GTK réellement utilisés et les choix techniques effectués dans chaque module.

Les extraits de code présentés ici sont basés sur l'implémentation réelle du projet. Les listings mettent l'accent sur les portions significatives (vérifications, branchements, gestion mémoire, raccordement des signaux) afin de conserver une lecture fluide dans le PDF.

L'objectif de ce volume est donc double : (1) expliquer \emph{ce que fait} chaque fonction du wrapper et (2) expliciter \emph{comment} elle le fait, notamment en termes de compatibilité GTK, d'ownership GLib et de comportement par défaut lorsque la configuration est absente.
