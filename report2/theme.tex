\section{Module \texttt{theme}}

Le module \texttt{theme} centralise l'application du thème CSS de démonstration.

\subsection{Macros de Configuration}

\begin{docbox}{Aucune macro \texttt{\#define} de configuration publique}
Le fichier d'en-tête \texttt{theme.h} expose uniquement la fonction \texttt{apply\_demo\_theme}. Il n'y a pas de structure \texttt{*\_config} ni de macro \texttt{*\_CONFIG\_DEFAULT} dans ce module.
\end{docbox}

Le module \texttt{theme} est volontairement centré sur le comportement global de rendu plutôt que sur des instances de configuration locales. Cette approche évite la dispersion des styles dans plusieurs composants.

\subsection{Fonctions API}

\begin{lstlisting}[language=C]
void apply_demo_theme(void) {
    GtkCssProvider *provider = gtk_css_provider_new();
    const char *css =
        ".btn-primary {"
        "  background: linear-gradient(180deg, #1f8c7a, #176b5e);"
        "  color: #ffffff;"
        "}"
        ".btn-success {"
        "  background: linear-gradient(180deg, #3c8f28, #2f6f1f);"
        "  color: #ffffff;"
        "}"
        ".btn-danger {"
        "  background: linear-gradient(180deg, #b74040, #8e2f2f);"
        "  color: #ffffff;"
        "}";

    gtk_css_provider_load_from_string(provider, css);
    gtk_style_context_add_provider_for_display(
        gdk_display_get_default(),
        GTK_STYLE_PROVIDER(provider),
        GTK_STYLE_PROVIDER_PRIORITY_APPLICATION
    );
    g_object_unref(provider);
}
\end{lstlisting}

Ce module injecte des classes CSS applicatives, ensuite utilisées par les widgets (ex: boutons \texttt{primary/success/danger}).

La fonction \texttt{apply\_demo\_theme} centralise les règles visuelles et garantit une cohérence graphique sur toute l'application. Elle réduit aussi le risque d'erreurs liées à des styles CSS dupliqués.
