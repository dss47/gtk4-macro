\section{Module \texttt{date}}

Le module \texttt{date} encapsule \texttt{GtkCalendar} pour la sélection de dates.

\subsection{Macros de Configuration}

\begin{docbox}{Macro \texttt{CALENDAR\_CONFIG\_DEFAULT}}
\begin{lstlisting}[language=C]
#define CALENDAR_CONFIG_DEFAULT ((calendar_config){ \
    .selected_date = NULL, \
    .show_day_names = true, \
    .show_week_numbers = false, \
    .no_month_change = false, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
    .on_day_selected = NULL, \
    .user_data = NULL, \
})
\end{lstlisting}
\end{docbox}

Cette macro initialise le calendrier dans un état standard et sûr. Les callbacks et données utilisateur sont nuls tant qu'ils ne sont pas explicitement fournis. Cela évite les appels invalides pendant la navigation calendrier.

L'usage d'une macro par défaut facilite aussi la compatibilité entre versions GTK: la configuration de base reste valide même si certains réglages évoluent. Cela protège le code de haut niveau contre des regressions.

\subsection{Fonctions API}

\begin{lstlisting}[language=C]
GtkWidget *create_calendar(const calendar_config *config) {
    GtkWidget *calendar = gtk_calendar_new();

    if (config == NULL) {
        return calendar;
    }

    if (config->selected_date != NULL) {
#if GTK_CHECK_VERSION(4, 20, 0)
        gtk_calendar_set_date(GTK_CALENDAR(calendar), config->selected_date);
#else
        gtk_calendar_select_day(GTK_CALENDAR(calendar), config->selected_date);
#endif
    }

    gtk_calendar_set_show_day_names(GTK_CALENDAR(calendar), config->show_day_names);
    gtk_calendar_set_show_week_numbers(GTK_CALENDAR(calendar), config->show_week_numbers);
    gtk_calendar_set_show_heading(GTK_CALENDAR(calendar), !config->no_month_change);

    if (config->on_day_selected != NULL) {
        g_signal_connect(calendar, "day-selected", G_CALLBACK(config->on_day_selected), config->user_data);
    }

    return calendar;
}
\end{lstlisting}

La fonction gère proprement la compatibilité GTK selon la version disponible.

Elle centralise la sélection de date, la configuration d'affichage et le branchement du signal \texttt{day-selected}. Cette centralisation rend le comportement du widget calendrier plus prévisible dans toute l'application.
