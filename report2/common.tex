\section{Module \texttt{common}}

Le module \texttt{common} centralise l'application des styles transverses à tous les widgets (CSS, marges, alignements, expansion, visibilité, infobulles).

\subsection{Macros de Configuration}

\begin{docbox}{Macro \texttt{WIDGET\_STYLE\_CONFIG\_DEFAULT}}
\begin{lstlisting}[language=C]
#define WIDGET_STYLE_CONFIG_DEFAULT ((widget_style_config){0})
\end{lstlisting}
\end{docbox}

Cette macro force une initialisation complète à zéro de la structure de style. En C, cela signifie des pointeurs à \texttt{NULL} et des flags booléens à \texttt{false}. Ce choix évite les lectures de mémoire non initialisée et réduit les plantages aléatoires.

Cette convention est essentielle pour éviter des erreurs de segmentation quand des champs optionnels ne sont pas renseignés. Elle fournit un contrat clair: tout champ non modifié reste dans un état neutre.

\subsection{Fonctions API}

\begin{lstlisting}[language=C]
void apply_widget_style(GtkWidget *widget, const widget_style_config *style) {
    if (widget == NULL || style == NULL) {
        return;
    }

    if (style->css_class && style->css_class[0] != '\0') {
        gtk_widget_add_css_class(widget, style->css_class);
    }

    gtk_widget_set_margin_top(widget, style->margin_top);
    gtk_widget_set_margin_bottom(widget, style->margin_bottom);
    gtk_widget_set_margin_start(widget, style->margin_start);
    gtk_widget_set_margin_end(widget, style->margin_end);

    if (style->set_halign) gtk_widget_set_halign(widget, style->halign);
    if (style->set_valign) gtk_widget_set_valign(widget, style->valign);
    if (style->set_hexpand) gtk_widget_set_hexpand(widget, style->hexpand);
    if (style->set_vexpand) gtk_widget_set_vexpand(widget, style->vexpand);
}
\end{lstlisting}

Cette fonction sert d'adaptateur entre la structure de configuration et l'API GTK. Elle applique uniquement les champs explicitement demandés pour préserver le comportement attendu des widgets.

Sur le plan d'architecture, \texttt{apply\_widget\_style} factorise la logique transversale pour tous les composants. Cela réduit la duplication de code et garantit une cohérence visuelle globale dans l'application.
