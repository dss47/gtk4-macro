\section{Module \texttt{display}}

Le module \texttt{display} gère les widgets d'affichage : progression, indicateur d'activité, texte et image.

\subsection{Macros de Configuration}

\begin{docbox}{Macros \texttt{PROGRESS\_BAR\_CONFIG\_DEFAULT}, \texttt{SPINNER\_CONFIG\_DEFAULT}, \texttt{LABEL\_CONFIG\_DEFAULT}, \texttt{IMAGE\_CONFIG\_DEFAULT}}
\begin{lstlisting}[language=C]
#define PROGRESS_BAR_CONFIG_DEFAULT ((progress_bar_config){ \
    .fraction = 0.0, \
    .text = NULL, \
    .orientation = GTK_ORIENTATION_HORIZONTAL, \
    .show_text = false, \
    .inverted = false, \
    .pulse_once = false, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
})

#define SPINNER_CONFIG_DEFAULT ((spinner_config){ \
    .size = 0, \
    .spinning = false, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
})

#define LABEL_CONFIG_DEFAULT ((label_config){ \
    .text = NULL, \
    .selectable = false, \
    .wrap = false, \
    .ellipsize = PANGO_ELLIPSIZE_NONE, \
    .max_width_chars = 0, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
})

#define IMAGE_CONFIG_DEFAULT ((image_config){ \
    .icon_name = NULL, \
    .file_path = NULL, \
    .pixel_size = 0, \
    .keep_aspect = true, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
})
\end{lstlisting}
\end{docbox}

Ces macros sécurisent l'initialisation des widgets d'affichage avec des valeurs neutres. Les pointeurs \texttt{NULL} empêchent les lectures de chaînes invalides, et les booléens explicites stabilisent le rendu par défaut. Cela limite les erreurs mémoire lors des premiers essais d'intégration.

Les macros d'affichage permettent également de standardiser le comportement visuel entre modules: une image sans chemin reste valide, une barre de progression sans texte reste fonctionnelle. Cette robustesse évite des plantages lors de données incomplètes.

\subsection{Fonctions API}

\begin{lstlisting}[language=C]
GtkWidget *create_progress_bar(const progress_bar_config *config) {
    GtkWidget *progress = gtk_progress_bar_new();
    if (config == NULL) return progress;

    gtk_progress_bar_set_fraction(GTK_PROGRESS_BAR(progress), config->fraction);
    gtk_progress_bar_set_show_text(GTK_PROGRESS_BAR(progress), config->show_text);
    gtk_orientable_set_orientation(GTK_ORIENTABLE(progress), config->orientation);
    if (config->text != NULL) {
        gtk_progress_bar_set_text(GTK_PROGRESS_BAR(progress), config->text);
    }
    apply_widget_style(progress, &config->style);
    return progress;
}

GtkWidget *create_image(const image_config *config) {
    GtkWidget *widget;
    if (config == NULL) return gtk_image_new();

    if (config->file_path != NULL && config->file_path[0] != '\0') {
        GtkWidget *pic = gtk_picture_new_for_filename(config->file_path);
        gtk_picture_set_content_fit(GTK_PICTURE(pic),
            config->keep_aspect ? GTK_CONTENT_FIT_CONTAIN : GTK_CONTENT_FIT_FILL);
        widget = pic;
    } else {
        widget = gtk_image_new_from_icon_name(config->icon_name);
    }

    apply_widget_style(widget, &config->style);
    return widget;
}
\end{lstlisting}

Le module choisit dynamiquement la meilleure représentation visuelle (icône système ou image fichier) en conservant une API homogène.

La fonction \texttt{create\_image} illustre bien l'approche du wrapper: masquer la complexité GTK derrière une interface simple. Le code applicatif choisit seulement la source (fichier ou icône) et le module gère le reste.
