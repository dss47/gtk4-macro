\section{Module \texttt{button}}

Le module \texttt{button} encapsule la création des boutons d'action et des boutons radio, avec options de style, icône et callbacks.

\subsection{Macros de Configuration}

\begin{docbox}{Macros \texttt{BUTTON\_CONFIG\_DEFAULT} et \texttt{RADIO\_BUTTON\_CONFIG\_DEFAULT}}
\begin{lstlisting}[language=C]
#define BUTTON_CONFIG_DEFAULT ((button_config){ \
    .label = NULL, \
    .icon_name = NULL, \
    .theme_variant = NULL, \
    .use_underline = false, \
    .has_frame = true, \
    .can_shrink = true, \
    .icon_size = 0, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
    .on_clicked = NULL, \
    .user_data = NULL, \
})

#define RADIO_BUTTON_CONFIG_DEFAULT ((radio_button_config){ \
    .label = NULL, \
    .group_with = NULL, \
    .is_active = false, \
    .style = WIDGET_STYLE_CONFIG_DEFAULT, \
    .on_toggled = NULL, \
    .user_data = NULL, \
})
\end{lstlisting}
\end{docbox}

Ces macros initialisent toutes les options dans un état prévisible. Les pointeurs \texttt{NULL} évitent les accès mémoire invalides, et les callbacks nuls empêchent les appels de fonction indéfinis. Cela réduit fortement les risques de segmentation fault dans les chemins de création.

L'intérêt pratique est de pouvoir écrire un code d'assemblage très compact: on initialise avec la macro, puis on personnalise seulement le label, l'icône et le callback. Cette méthode diminue les oublis de champs et stabilise l'interface.

\subsection{Fonctions API}

\begin{lstlisting}[language=C]
GtkWidget *create_button(const button_config *config) {
    GtkWidget *button;
    const char *label;

    if (config == NULL) {
        return gtk_button_new();
    }

    label = config->label ? config->label : "";
    button = config->use_underline ? gtk_button_new_with_mnemonic(label)
                                   : gtk_button_new_with_label(label);

    gtk_button_set_has_frame(GTK_BUTTON(button), config->has_frame);
    gtk_button_set_can_shrink(GTK_BUTTON(button), config->can_shrink);

    apply_theme_variant(button, config->theme_variant);
    apply_widget_style(button, &config->style);

    if (config->on_clicked) {
        g_signal_connect(button, "clicked", G_CALLBACK(config->on_clicked), config->user_data);
    }

    return button;
}
\end{lstlisting}

La fonction construit un bouton utilisable même en cas de configuration partielle. Elle ajoute ensuite les options visuelles et connecte le callback si fourni.

La fonction \texttt{create\_button} joue aussi un rôle d'adaptation entre des besoins métiers simples et l'API GTK plus verbeuse. En encapsulant les variantes de thème et le branchement du signal \texttt{clicked}, elle rend le code applicatif plus lisible.
